# 每日自动日报技能

## 概述

本技能实现：用户在指定session随意聊工作 → 自动记录 → 定时生成结构化日报 → 自动发送到指定渠道。全程无需手动整理排版。

## 文件结构

```
daily-report-auto/
├── SKILL.md
├── references/
│   ├── config-guide.md          # 首次配置全流程引导
│   ├── interaction-guide.md     # 日常使用交互规范
│   ├── report-templates.md      # 多角色日报模板

└── scripts/
```

## 数据存储约定

所有数据文件存储在 workspace 下：

- `daily-report/config.json` — 渠道、定时、模板、角色配置
- `daily-report/work-records/YYYY-MM-DD.md` — 当日工作记录
- `daily-report/reports/YYYY-MM-DD.md` — 当日最终日报

## 核心工作流

### 0. 全渠道专属快捷触发
- 用户配置任意渠道（飞书/微信/OpenClaw）的日报功能时，系统自动为对应渠道创建全新的永久独立日报会话，与主会话（agent:main:main）完全隔离，互不干扰全局记忆，统一设置会话标签为「daily-report-session」，会话键由系统按规则自动生成无需手动指定
- 全渠道通用触发词：【日报】，任意会话输入「日报」即可直接进入对应渠道的专属日报独立会话，历史记录完整保留；输入「退出日报」即可跳出当前日报专属独立子会话，自动返回之前的原对话页面，所有已录入工作记录自动保存
- 触发词支持用户自定义修改
- 用户也可直接在会话列表中点击对应渠道标注为「daily-report-session」的专属子会话条目进入
- 独立会话内所有输入默认自动作为工作内容记录，无需额外切换模式，关闭子会话即可自动返回原对话，所有记录自动同步保存

### 1. 首次配置（用户说"日报怎么用"等触发）

读取 [references/config-guide.md](references/config-guide.md)，按步骤引导用户完成：
1. 功能说明
2. 自动检测当前已配置的可用渠道（飞书/微信/OpenClaw），若存在已连接的渠道则直接使用当前渠道，无需用户选择；若无可使用渠道则引导用户选择专属session渠道（飞书/微信/OpenClaw）
3. 按渠道引导用户提供基础信息：
   - 飞书：仅需App ID/APP Secret 2项，无需任何技术配置知识，告知用户将自动创建飞书专属日报独立会话，通用触发命令为「日报」，支持后续自定义修改
   - 微信：引导完成微信账号绑定，告知用户将自动创建微信专属日报独立会话，通用触发命令为「日报」，支持后续自定义修改
   - OpenClaw：引导用户确认OpenClaw专属触发命令（默认值：「日报」，支持自定义修改），同时告知用户将为其创建独立的专属日报子会话
4. **自动完成渠道全路径配置：**
   - 通用：调用对应渠道setup技能，自动配置收发消息能力、校验权限、获取会话标识、发送测试消息
   - 全渠道通用：自动为对应渠道生成独立日报会话，设置标签为「daily-report-session」，同时配置通用触发命令「日报」，退出命令「退出日报」，所有触发词均支持自定义修改，无需绑定当前配置会话，独立会话历史记录永久保留
5. 设置定时时间（记录提醒/草稿推送/日报发送，支持默认一键确认）
6. 选择角色 + 确认日报模板（支持默认一键确认）
6.1 询问用户是否开启「主会话智能提取自动生成日报」功能，说明功能可自动从主会话识别工作内容生成日报，无需手动录入，支持默认一键开启/关闭
7. 自动写入配置、创建定时任务、完成开通确认：
   - 对应渠道额外告知用户两种进入方式：① 任意会话输入通用触发词「日报」即可直接跳转进入对应渠道的日报独立会话，历史记录完整保留，输入「退出日报」即可返回原对话；② 也可在会话列表中点击对应渠道的「daily-report-session」子会话条目进入，无需手动切换session

全程用户仅需提供1-2项基础信息+确认默认选项，无任何技术门槛，所有配置操作自动完成，不等待用户反问。

### 2. 工作记录

- 记录范围包含：用户在日报专属session发送的消息，以及主会话当日的工作相关内容（主会话提取功能开启时）
- 自动识别工作相关内容，智能分类：工作内容、成果、问题、计划
- 追加写入 `daily-report/work-records/YYYY-MM-DD.md`
- 支持用户主动修改、删除记录条目
- 16:00草稿推送后，用户发送的修改、补充内容（支持文本、文件格式）将自动整合到原始记录中，进行优化调整，最终生成优化后的日报内容

### 3. 日报生成

- 16:00定时推送当日工作记录草稿，供用户确认、修改或补充内容（支持文本、文件格式）
- 整合用户在草稿阶段补充/修改的所有内容，与已有的工作记录（包含主会话提取内容+日报会话记录内容）合并
- 按用户确认的模板结构化排版，自动润色优化汇报话术，补全规范表达
- 生成最终优化后的日报到 `daily-report/reports/YYYY-MM-DD.md`
- 17:00按配置渠道自动发送最终优化后的日报内容

### 4. 日报发送

- 按配置渠道发送最终日报
- 复用 feishu-messaging 等渠道技能
- 发送成功/失败均有明确反馈

### 5. 定时任务（通过 cron 工具实现）

配置完成后创建 3 个 cron job：
- 记录提醒（默认 09:00）
- 草稿推送（默认 16:00）
- 日报发送（默认 17:00）

### 6. 智能判断

- 周末/法定节假日自动跳过，发送休息提示
- 用户说"今天请假"则取消当日日报
- 临时改时间仅当日生效，次日恢复默认

### 7. 主会话智能提取自动生成日报（可选开启）
> 功能触发时间完全复用用户之前配置的日报发送时间（无需重复设置）
> ⚠️ 优先级说明：日报独立子会话内手动录入的内容为核心工作记录，主会话提取的工作内容仅作为补充信息
1. 定时触发：到用户配置的日报发送时间时，自动触发当前渠道的日报独立子会话执行生成逻辑
2. 日志截取范围：仅读取【当日00:00 - 配置的发送时间】的全局主会话日志，发送时间之后的内容不纳入当日日报，自动计入次日统计
3. 内容过滤规则：自动识别主会话内容，仅保留工作任务、成果、待办相关内容；过滤命令交互、闲聊、无实质工作内容的对话，自动提炼为「今日完成（补充）」「待跟进（补充）」模块，与子会话核心记录合并展示
4. 推送逻辑：
   - 若当日有任意有效工作内容（子会话手动录入为核心 + 主会话提取为补充） → 自动生成完整日报（优先展示子会话核心内容，补充内容标注来源）并推送给用户
   - 若当日无任何有效工作内容 → 推送极简提示「今日无有效工作记录」，避免空内容推送
5. 存储规则：自动生成的日报完整内容自动存入对应渠道的日报子会话，历史记录永久可追溯，次日自动复用同一会话更新

## 日常操作命令

| 用户输入 | 动作 |
|---------|------|
| 帮我生成今日日报 | 整合记录，生成日报草稿 |
| 查看今日工作记录 | 展示当日已记录内容 |
| 重发今日日报 | 重新发送到配置渠道 |
| 修改记录（具体内容） | 更新对应记录条目 |
| 删除记录（具体内容） | 删除对应记录条目 |
| 补充记录（具体内容/文件） | 整合用户补充的文本/文件内容到当日工作记录，优化日报内容 |
| 今天请假不发日报 | 取消当日自动发送 |
| 修改日报模板 | 进入模板编辑模式 |
| 今天日报XX:XX发送 | 临时修改当日发送时间 |
| 开启/关闭主会话自动提取日报 | 切换是否启用从主会话自动提取内容生成日报的功能 |
| 日报 / 会话列表点击对应渠道「daily-report-session」 | 【全渠道通用】输入通用触发词「日报」直接进入对应渠道的独立日报子会话，所有触发词均支持自定义修改 |
| 退出日报 / /exit | 【全渠道通用】跳出当前日报专属独立子会话，自动返回之前的原对话页面，所有已录入工作记录自动保存 |
| 关闭子会话页面 | OpenClaw专属：关闭日报子会话自动返回原对话，记录自动保存 |

## 渠道引导语切换

按 `config.json` 中的 `channel` 字段自动切换提醒/确认话术中的渠道名称。详见 [references/interaction-guide.md](references/interaction-guide.md)。
