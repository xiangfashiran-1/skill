# 首次配置全流程引导

## 触发词

以下输入触发首次配置：日报怎么用、如何使用日报生成、日报功能怎么开启、帮我开通日报、配置日报

## 环节1：功能说明

发送以下说明：

> 你好～我是 daily-report-auto 每日自动日报助手，核心功能：
> 1. 你在指定的飞书/微信/OpenClaw对话里随意聊工作，我会自动记录
> 2. 按你设定的时间提醒记录、推送草稿、自动发送最终日报
> 3. 支持按岗位角色选择模板，自动结构化排版
> 4. 全程自动发送到你指定的会话，无需手动复制粘贴
>
> 接下来我会一步步带你完成配置，马上就能启用～

## 环节2：自动检测渠道
自动检测当前已连接的可用渠道（飞书/微信/OpenClaw）：
- 若检测到已配置的渠道，直接使用当前渠道，无需用户选择，提示「已自动检测到当前使用【XX】渠道，将直接复用作为日报专属通道」
- 若未检测到可用渠道，再引导用户选择专属session渠道：飞书 / 微信 / OpenClaw

## 环节3：按渠道配置

### 飞书session
首先检测当前是否已存在全局飞书channel配置：
1. 无全局飞书配置：引导用户提供App ID和APP Secret，自动调用feishu-setup完成全局飞书机器人配置，作为日报专属通道
2. 已有全局飞书配置：给用户两个选择：
   - 选择1：复用现有全局飞书机器人，不需要额外配置，直接开始使用
   - 选择2：创建独立专属日报机器人（推荐，和通用对话机器人完全独立隔离），仅需提供新机器人的App ID和APP Secret，后续全部自动完成：
     ✅ 自动部署独立轻量后台监听服务，和全局飞书通道完全隔离互不干扰
     ✅ 自动开通收消息+发消息全路径权限
     ✅ 自动发送测试消息，引导用户回复任意内容自动获取会话标识
     ✅ 自动完成所有配置校验，无需用户手动填写open_id/chat_id/其他配置项
用户仅需按指引复制App ID和APP Secret 2个参数即可，全程无技术门槛。

### 微信session

需要 3 项信息：
1. 微信 AppID — 微信开放平台/企业微信后台获取
2. 微信 AppSecret — 同上
3. 会话标识：
   - 单聊：用户 openid
   - 群聊：群聊 id
   - 给机器人发消息自动返回当前会话标识

### OpenClaw session

需要 2 项信息：
1. OpenClaw 对接密钥 — OpenClaw 后台 → 渠道配置 → 密钥
2. OpenClaw 对话ID（Session ID），获取方式任选：
   - 在对话里发送"请返回我的对话ID"
   - 命令行：`openclaw sessions list`
   - 文件：`~/.openclaw/agents/main/sessions/sessions.json`

### 校验

配置信息收集完毕后：
- 飞书渠道：自动调用feishu-setup完成机器人websocket连接配置、权限校验、双向交互开通，发送测试消息到你给机器人发消息的会话中，自动获取目标会话标识
- 其他渠道：尝试向目标session发送测试消息
- 成功 → 提示"【XXX】配置校验成功，测试消息已发送到你的专属session！"
- 失败 → 提示具体错误，引导修正

## 环节4：全局快捷触发功能独立确认（默认关闭，需用户主动确认开启）
> 📲 现在和你确认是否开通【全局快捷触发功能】：
> ✅ 功能说明：开通后你在当前渠道**任意会话（包括群聊、其他单聊）**输入统一触发词「日报」就能直接跳转到日报专属独立会话，输入「退出日报」即可自动返回原对话，无需专门查找机器人
> ❌ 不开通的话，仅能直接和日报专属机器人/会话聊天记录工作
> 请明确回复：开通 / 不开通

用户确认后记录到配置。

## 环节5：配置定时时间

> 请设置 3 个核心时间（可直接使用默认）：
> - 时间点1（记录提醒）：09:00
> - 时间点2（草稿推送）：16:00
> - 时间点3（日报发送）：17:00
>
> 修改格式示例：时间点1=09:30；时间点2=16:30；时间点3=17:30

用户确认后记录到配置。

## 环节6：选择角色 + 确认日报模板

> 请选择你的岗位角色，自动匹配模板：
> 1. 产品经理
> 2. 研发工程师
> 3. 测试工程师
> 4. 运营
> 5. UI/UX设计
> 6. 行政/人事
> 7. 自定义模板

展示对应模板（详见 [report-templates.md](report-templates.md)），支持多轮修改，用户明确确认后：

> ✅ 日报模板已最终确认，后续将按此模板生成日报。

## 环节7：写入配置 + 创建定时任务

将所有配置写入 `daily-report/config.json`，格式：

```json
{
  "channel": "feishu|wechat|openclaw",
  "channelConfig": {
    "appId": "...",
    "appSecret": "...",
    "targetId": "..."
  },
  "schedule": {
    "remind": "09:00",
    "draft": "16:00",
    "send": "17:00"
  },
  "permissions": {
    "shortcutsEnabled": false,
    "autoExtractEnabled": false,
    "mainSessionExtractEnabled": false
  },
  "role": "研发工程师",
  "template": ["今日开发内容", "联调/自测情况", "Bug修复", "阻塞问题", "明日计划"],
  "timezone": "Asia/Shanghai",
  "enabled": true
}
```

使用 cron 工具创建 3 个定时任务（agentTurn 类型，isolated session）：
- 记录提醒 cron
- 草稿推送 cron
- 日报发送 cron

每个 cron 的 message 中包含具体指令，如"读取 daily-report-auto 技能，执行记录提醒流程"。

> 🎉 你的 daily-report-auto 功能已全部配置完毕并正式启用！
> 后续所有记录、提醒、日报都将发送到你的【XXX】session。
>
> 📌 已开通功能说明：
> - 全局快捷触发：{{已开通/未开通}}，触发词：{{对应渠道触发词}}
> - 数据智能提取：{{已开通/未开通}}，主会话自动提取：{{已开通/未开通}}
> - 日报会话内输入 `/exit` 即可随时退出，自动返回原对话页面，所有记录自动保存
