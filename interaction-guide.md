# 日常使用交互规范

## 定时触发交互

### 记录提醒（按渠道切换）

- 飞书：早上好～已到工作记录时间，请在**飞书**本会话聊工作，我会自动记录。
- 微信：早上好～已到工作记录时间，请在**微信**本会话聊工作，我会自动记录。
- OpenClaw：早上好～已到工作记录时间，请在**OpenClaw**本会话聊工作，我会自动记录。

### 草稿推送

> 温馨提醒：为你整理了今日工作内容草稿（含本地日志+本session沟通记录）：
> 1. [记录条目1]
> 2. [记录条目2]
> 3. [记录条目3]
>
> 如需补充/修改，在本session告诉我即可。

### 日报发送成功

> ✅ 今日日报已自动发送至本session，内容预览：
> 【今日完成】...
> 【遇到问题】...
> 【明日计划】...

## 用户主动操作

### 生成日报

用户：帮我生成今日日报
→ 整合 `work-records/YYYY-MM-DD.md`，按模板生成，展示草稿
→ 用户确认后发送，或修改后再发送

### 查看记录

用户：查看今日工作记录
→ 读取并展示 `work-records/YYYY-MM-DD.md` 内容

### 重发日报

用户：重发今日日报
→ 读取 `reports/YYYY-MM-DD.md`，重新发送到配置渠道

### 修改记录

用户：把"XXX"改成"YYY"
→ 编辑 `work-records/YYYY-MM-DD.md` 对应条目
→ 展示修改前后对比

### 删除记录

用户：删除"XXX"这条
→ 从 `work-records/YYYY-MM-DD.md` 删除
→ 展示剩余记录

### 请假

用户：今天请假不发日报
→ 取消当日所有定时任务
→ 提示"已取消今日自动日报，明天恢复正常推送"

### 修改模板

用户：修改日报模板
→ 展示当前模板，进入编辑模式
→ 用户确认后更新 `config.json`

### 临时改时间

用户：今天日报18:00发送
→ 修改当日 cron 发送时间
→ 提示"已将今日日报发送时间改为18:00，次日将恢复默认"

## 工作内容自动识别规则

从用户消息中识别工作相关内容时：
1. 包含动词+名词的工作描述（如"完成了XX"、"修复了XX"、"参加了XX会议"）
2. 包含项目/任务关键词
3. 排除纯闲聊、问候、表情

分类规则：
- **今日完成**：已完成的工作、交付物
- **遇到问题**：阻塞、bug、异常、困难
- **明日计划**：计划、打算、准备做的事
- **其他**：不确定分类的记录原文

## 发送渠道切换

所有提醒/确认话术中的渠道名称，根据 `config.json` 中 `channel` 字段自动替换：
- `feishu` → 飞书
- `wechat` → 微信
- `openclaw` → OpenClaw
